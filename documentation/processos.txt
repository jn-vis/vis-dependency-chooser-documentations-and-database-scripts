		O que acontece quando um currículo é salvo? - Vis.1
		1) O currículo, juntamente com seus hashes é salvo - Vis.1.1 [OK]
		2) O arquivo do currículo é salvo no bucket - Vis.1.2
		3) O currículo é enviado para vagas aderentes agendadas como instantâneas - Vis.1.3 [OK]
		4) Notificar recrutador caso este não tenha saldo para receber currículo imediato. - Vis.1.4
		5) Debita-se saldo de recrutador  - Vis.1.5

		O que acontece quando uma vaga é salva?
		1) A vaga, juntamente com seus hashes é salva [OK]
		2) Gera-se uma lista de currículos
		3) Agenda-se envio de currículos para cada vaga de acordo com sua opção de agendamento escolhida [OK]

		O que acontece quando um agendamento de currículos é executado?
		1) Se traz para memória todos os recrutadores de todas as vagas marcadas para aquela opção de agendamento [OK]
		2) Elimina-se todas as vagas de todos os recrutadores sem saldo para execução agendada de envio de currículos, debitando saldos [OK]
		3) Notifica-se todos os recrutadores cujo os saldos são insuficientes para envio dos curriculos [OK]
		4) Busca-se todos os hashes de todas as vagas que sobraram após a filtragem anterior e com esses hashes, buscam-se os currículos [OK]
		5) Os currículos são enviados para cada recrutador

		O que acontece quando um currículo é visualizado?
		1) Valida-se aderência entre vaga e curriculo
		2) Transfere-se saldo do recrutador para o candidato
		3) Se faz o download do currículo
		4) Registra-se a visualização do curriculo
